// THE MATRIX
A 6×6 grid of all 26 letters (A–Z) and digits 0–9. Each character has a unique coordinate – a row number and a column number. The matrix is the core lookup table the cipher uses to convert letters into number pairs and back. Its layout is determined entirely by your Grid Key.
// GRID KEY
A word or phrase that shuffles the order of characters in the matrix. The same Grid Key always produces the same matrix. Change it and the whole grid rearranges – the same message will encrypt completely differently. Share this with your recipient so they can build the same grid to decrypt.
// TRANSPOSITION KEY
A word used to scramble the coordinate stream before final lookup. After each character is turned into a row/column pair, all the rows and columns are shuffled by reading them in the alphabetical order of this key's letters. Longer keys = harder to crack. If your key is too short for your message, the app will auto-extend it (see AUTO-FIX below).
// SHIFT SEQUENCE
A string of digits (e.g. 719) that shifts each coordinate by a cycling amount before transposition. Each digit shifts the next coordinate by that many steps around the grid. More digits = more variation in the shift pattern = stronger cipher. Zeros are ignored – use digits 1–9.
// KEY STRENGTH DOTS
Each key field shows 4 dots below it. Dots fill left to right as your key gets stronger – red = weak, amber = moderate, green = strong, bright green = void-tier. Grid Key scores on unique characters, Transposition Key on letter count, Shift Sequence on digit count.
// NULL PAD (X)
The cipher processes characters in pairs. If your message has an odd number of valid characters, the last one has no partner and gets dropped. Turning on NULL PAD automatically appends an X to make the count even, so no characters are lost. The recipient will see the X at the end and can ignore it.
// MULTI-PASS
Set PASSES (1–4) to run the cipher multiple times in sequence using the same three keys. Each pass feeds the previous ciphertext back in as input. Double-pass or higher significantly increases diffusion. To decrypt, set the same pass count – the app reverses all passes automatically.
// ENCRYPTING A MESSAGE
1. Set your three keys (or hit ROLL for random ones).
2. Type your message in the PLAINTEXT box.
3. Hit ENCRYPT or press Ctrl+Enter.
4. The ciphertext appears below. Hit COPY to copy it.
5. Hit SHARE to generate a blob containing keys + ciphertext in one string – copy and send that to your recipient.
// DECRYPTING A MESSAGE
Option A – Manual: Set the same three keys used to encrypt, switch to DEC mode, paste the ciphertext, hit DECRYPT.

Option B – Share blob: Paste the VMC-MSG:... string into the field at the top and hit LOAD. The app fills in all keys and decrypts automatically.
// SHARE & LOAD
After encrypting, hit SHARE. This bundles your Grid Key, Transposition Key, Shift Sequence, pass count, and the ciphertext into a single encoded string starting with VMC-MSG: – copy and send it. The recipient pastes it into the VMC-MSG field and hits LOAD. Keys are filled automatically and the message is decrypted instantly. No manual key entry needed.
// QR CODE
After encrypting, hit QR to generate a scannable QR code containing the full VMC-MSG blob. The recipient scans it on another device, pastes the decoded string into the VMC-MSG field, and hits LOAD. QR is limited to ~2953 bytes – use shorter messages or SHARE for longer ones.
// AUTO-FIX: KEY TOO SHORT
If your Transposition Key is too short relative to your message length, repeating patterns can appear in the ciphertext. The app detects this automatically and extends the key using a deterministic algorithm derived from your original key – so the recipient can reproduce the same extension. The Transposition field updates to show the extended key. A yellow banner explains what changed. The SHARE blob always contains the actual key that was used.
// AUTO-VERIFY
After every encryption, the app silently decrypts the ciphertext using the current keys and compares the result to your original input. If it shows ✓ ROUND-TRIP VERIFIED, your recipient will be able to decrypt successfully. If it shows ✗ VERIFY FAILED, your keys may have changed since encryption – re-encrypt before sending.
// INDEX OF COINCIDENCE (IC)
The IC meter appears below the output after each run. It measures character frequency distribution in the ciphertext. FLAT (near 0.0278) means the output looks statistically random – good. MIXED means partial structure is present. STRUCT means strong patterns remain, which may indicate weak keys or too few passes.
// FREQ ANALYSIS
Open the FREQ ANALYSIS panel after any run to see a bar chart of character distribution in the output, sorted by frequency. A flat, even spread indicates good cipher strength. Tall dominant bars suggest patterns worth addressing – try longer keys or more passes.
// CRACK DIFFICULTY
Open CRACK DIFFICULTY after any run to see the estimated keyspace of your current key combination, broken down by grid permutations, transposition permutations, and shift combinations. Rating: WEAK / MODERATE / STRONG / VOID. Longer keys in all three fields push toward VOID.
// STEP-THROUGH
After any encryption or decryption, open the STEP-THROUGH panel and hit PLAY to watch the cipher process one character at a time. Blue cells in the matrix show where the input character lives. Green cells show where the output character comes from. Use the SPD slider to control playback speed. Hit RESET to replay from the start.
// GROUP 5 & EXPORT
After encrypting, two buttons appear below the output. GROUP 5 splits the ciphertext into space-separated 5-character blocks – traditional tradecraft format, easier to read and transcribe. EXPORT downloads a .txt dossier containing your keys, IC score, crack rating, full ciphertext, and the VMC-MSG blob for archiving or offline handoff.
// MESSAGE HISTORY
The last 5 encryptions are stored in MESSAGE HISTORY for the current session only – nothing persists after you close the tab. Each entry shows timestamp, plaintext preview, keys used, and ciphertext preview. Hit ↩ USE on any entry to reload its keys and plaintext into the cipher fields.
