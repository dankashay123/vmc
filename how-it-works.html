How It Works

A plain-language breakdown of the cipher — no math degree required.

1 — THE GRID
The 6×6 Matrix

The foundation of Void Matrix is a 6-by-6 grid containing all 26 letters of the alphabet (A–Z, with I and J treated as distinct characters) plus the 10 digits 0–9. That's exactly 36 characters filling 36 squares.

Every character in the grid has a unique address — a row number and a column number. To encrypt the letter S, you find it in the grid and note its coordinates. That coordinate pair becomes the basis for everything else.

1 2 3 4 5 6 1 O B S C U R 2 E A D F G H 3 I J K L M N 4 P Q T V W X 5 Y Z 0 1 2 3 6 4 5 6 7 8 9

In this grid, the letter S is at Row 1, Column 3. The letter E is at Row 2, Column 1. Every character has exactly one address, and every address holds exactly one character.

2 — THE KEYS
Three Keys, Three Jobs

The cipher uses three independent keys. Each one does a different job. All three are required to decrypt.

KEY 01
GRID KEY
Shuffles the layout of the entire matrix. A word like OBSCURE pushes its letters to the front of the grid, displacing everything else. Change this key and the whole grid rearranges — the same message encrypts completely differently.
KEY 02
TRANSPOSITION KEY
Scrambles the order of coordinates after they've been extracted. The coordinates are written in columns under the key's letters, then read back in alphabetical order. Longer keys scramble more aggressively.
KEY 03
SHIFT SEQUENCE
A string of digits (1–9) that shifts each coordinate by a cycling amount before transposition. The shift pattern cycles through the digits repeatedly, so every position in the message shifts by a different amount.
3 — ENCRYPTION
Step by Step
1
FRACTIONATION
Each character in your message is looked up in the grid and converted to a row/column coordinate pair. A 5-character message produces 5 row values and 5 column values — 10 numbers total.
MESSAGE: HELLO H → (2,6)   E → (2,1)   L → (3,4)   L → (3,4)   O → (1,1)
2
SHIFT
Each coordinate is shifted by the next digit in the Shift Sequence (cycling back to the start when it runs out). The shift wraps around the grid — going past 6 loops back to 1. All rows are shifted first, then all columns.
SHIFT SEQUENCE: 719 Row values: 2 2 3 3 1 + shifts 7 1 9 7 1 → 3 3 6 4 2 Col values: 6 1 4 4 1 + shifts 9 7 1 9 7 → 3 2 5 7→1 2
3
TRANSPOSITION
All shifted rows and columns are joined into one long sequence and written in columns under the letters of the Transposition Key. The columns are then read back in the alphabetical order of the key's letters — completely scrambling the sequence.
TRANSPOSITION KEY: VOID V   O   I   D ─   ─   ─   ─ 3   3   6   4 ← shifted rows 2   3   2   1 ← more values ... Read back in order: D I O V → scrambled sequence
4
RECOMBINATION
The scrambled sequence is read back as coordinate pairs (first number = row, second = column) and each pair is looked up in the grid. Whatever character lives at that address becomes part of the ciphertext.
FINAL LOOKUP Scrambled pair (4,2) → grid lookup → Q Scrambled pair (1,5) → grid lookup → U ...and so on until the ciphertext is complete
4 — PASSES
Running It Multiple Times

A single pass through the cipher is reasonably strong. But Void Matrix lets you run 1–4 passes. Each pass takes the ciphertext from the previous pass and feeds it back in as input, using the same three keys.

This is called diffusion — each additional pass spreads patterns further and makes frequency analysis increasingly useless. Two passes is a significant jump in difficulty. Four passes is effectively impractical to brute-force.

To decrypt, the same pass count must be used in reverse. The app handles this automatically — set the same number of passes and hit decrypt.

Important: the pass count is encoded into the VMC-MSG share blob automatically. If you load a blob, the app sets the correct pass count for you.

5 — DECRYPTION
Running It Backwards

Decryption is every step in reverse. The ciphertext characters are looked up in the grid to recover coordinates, the transposition is un-scrambled, the shifts are subtracted instead of added, and the resulting coordinates map back to the original characters.

The app handles all of this automatically. All you need are the same three keys and the same pass count used to encrypt.

6 — SECURITY
Why It's Hard to Break

Three separate keyspaces have to be searched simultaneously to crack this cipher:

Grid Key permutations:    36! ≈ 3.7 × 10⁴¹ Transposition perms:     N! where N = key length Shift combinations:      10^(digit count) Combined keyspace:       effectively astronomical

Even knowing the algorithm completely — even having the source code — an attacker still needs all three keys. The cipher's strength comes from the keys, not from secrecy about how it works. This is a well-established principle in cryptography called Kerckhoffs's principle.

That said: Void Matrix is a classical cipher, not a modern cryptographic primitive like AES. It's not recommended for protecting state secrets. It's designed for personal use — secure communication between people who share keys, with a strong emphasis on usability and aesthetic.

7 — FAQ
Common Questions
What happens if I use the wrong keys to decrypt?
You'll get garbled output — random-looking characters. The app will warn you if it detects no vowels in the decrypted text, which is a strong signal something is wrong.
Does my message leave my device?
No. All encryption and decryption runs entirely in your browser. No data is sent to any server. The app works completely offline after the first load.
What's the VMC-MSG blob?
It's a base64-encoded string containing your Grid Key, Transposition Key, Shift Sequence, pass count, and ciphertext bundled together. Your recipient pastes it into the app and loads it — all keys fill automatically and the message decrypts in one tap.
What does the Index of Coincidence tell me?
It measures how flat the character frequency distribution is in your ciphertext. A perfectly random cipher produces an IC near 0.0278. Higher values mean patterns exist — try longer keys or more passes.
Can I encrypt numbers and symbols?
Numbers 0–9 are part of the grid and encrypt normally. Symbols, spaces, and punctuation are stripped out before encryption — only A–Z and 0–9 are valid input characters.
